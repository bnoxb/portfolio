Horagotchi

My first real project. My cohort and I were given the task of creating our very own tamagotchi. A Tamagotchi was a 90's toy that helped kids learn about responsiblity. You must take care of it, feeding it, play with it, and put to sleep, so it can grow.

This was the first program that I had to design from scratch. I had to take a vision that I had for the project and design it to meet the requirements of the class project. I chose to base the aesthetics of my app from a sci-fi computer game called Endless Space 2. This project reaffirmed my new love for programming.

Tower-Battle

The Second project at GA. This project was intended to be a browser game of any type. I was in charge of every aspect of designing the game. To start off I had to pick something that would be challenging and allow me to learn a lot of new things. I wanted to get practise with classes in javascript and so I created Tower-Battle.

Tower-Battle is a game where the player can pick from three different unit types. The player must create units to walk across the game board and destroy the demon tower. I planned on making each unit its own object instantiated from different classes, that way it would be scaleable and simple for me to change stats, or even create new unit types. There is a "research" feature that allows the player to upgrade the stats of units. I learned a lot from this project, classes and how powerful they are. Proper planning is crucial before you start a project, so you can structure everything in the best way. This is especially important when you want your app to be scaleable.

This project was great because I had to explore JS on my own. Figuring out what techniques to use to accomplish my goals for this game. There was also a lot of complicated problems, specific to the game that had to be overcome. The few days I had to make this game were definetely a lot of fun!

Orderve

This is my third project at General Assembly, Orderve. For this project I was put into a team and we had to create an application that has a database and RESTful routes. We used EJS templates, Node.js, Express, Mongoose and MongoDB.

The concept we picked for this app: to match people looking for chefs, bartenders or servers for an event, with other people with those skills that are interested in making some extra money. So, the host of an event can post their event on the app, list what services they need, and people can make a “Bid” on the event.

Orderve was a challenging project. It was our first time making a database with models and routes. Also, we were trying to learn how to program with a new language. On top of all that, it was the most complex website we had made up to that point. We decided to implement an agile work environment. This was paramount to our success in our first team application. This project was awesome, I got valuable team experience, programming experience and created my first full-stack app!

TravlTrakr

Back-End Refactor

The fifth project in my journey through General Assembly is called TravlTrakr. This project was an interesting one, bringing lots of new challenges to overcome. For this I used React.js, SQL, Java and SpringBoot. The Idea for this project was to take another student’s project and refactor it to use Java Springboot on the back-end.

Getting a solid plan for this project was key. I had to make sure that I knew how everything connected in the app that I was assigned to refactor. I spent a lot of time reading through his code, understanding how everything was structured. Once I had a good understanding of the other students’ code I was able to begin rebuilding the backend. Some of the most challenging things I dealt with on this project were, making a third party API call using java, and refactoring some of his front-end to be able to work with a SQL database instead of a MongoDB. Accomplishing these tasks forced me to learn a lot about how Java and Springboot work to communicate with APIs.

Basically this app helps users plan their trips abroad by finding restaurants near where they are planning to go. The user inputs where they are traveling to, and the dates they will be there. After that a google map is loaded with markers showing the locations of all restaurants in the area, randomly choosing ones based on number of reviews, hot and new or simply the best in the area.

Condue

Condue, the fourth project while attending General Assembly. The scope of this project was to learn how to use React.js and integrate it with an Express back-end. I had to include a feature that has full CRUD functionality with an API that I created, and include a feature that makes a third party API request. I used React.js, Express, Mongoose, MongoDB, and Reactstrap for this project.

I chose to create a website, that a restaurant might have as their website. I created a fake restaurant called Condue, and its website you can make reservations, reviews and check the weather so you can plan a “Patio Day”. This project was challenging because I had to learn how to code an app that uses a language, which has an entirely different type of programming style: functional programming. The biggest challenge was learning how to pass data from one component to another further up, or down in the chain.

In the end, I learned a lot about React.js and continued to develop my programming skills. React is an amazing framework to work with and working with third party API’s is a lot of fun.

Condue:RE

For my last project at General Assembly I decided to add some new features to my previous project: Condue. I wanted to add some more functionality, just so I could get experience doing some cool tricks.

I wanted to have all the benefits of a user being logged in, without the user having to login. The best solution was to use a one click authentication. At this point I really wanted to learn about how to use a real time database. Lastly, I wanted to design my own calendar, mainly because I wanted to learn more about creating tools.

So for Condue:RE I added an interactive calendar that will let you make or check reservations, this depends on whether or not your logged in as a user or admin. To log in as a user, you just have to click login and you can use your google credentials. Lastly, I added a live chat feature in the About page. Anyone can make an admin account at this point to test it out.
